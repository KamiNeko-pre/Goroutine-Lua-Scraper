### 🚀 lua-spider: 3 周 MVP 极速突击清单 (4月28日 - 5月19日)

#### **🔥 Week 1：工程骨架与存储闭环（Day 1 - Day 7）**

**核心目标：** 快速搭好舞台。API 能接客，日志能排错，数据能存盘。

* **📚 理论弹药（严格对齐博客）：**
* `Go语言配置管理神器——Viper中文教程`
* `gin框架快速指南` & `gin框架路由拆分与注册`
* `gin框架中使用zap日志库`
* `GORM入门指南` & `GORM CRUD操作指南`


* **🛠️ 工程落地（每天干什么）：**
1. 建立 `configs/config.yaml`，用 Viper 在 `internal/config` 中完成结构体映射。
2. 在 `internal/logger` 初始化 Zap 全局日志。
3. 在 `internal/repository` 初始化 GORM MySQL 连接。设计一张极其简单的表 `github_repos`（存仓库名、Star数、描述）。
4. 在 `internal/controller` 暴露 `POST /api/v1/task`。接收前端 JSON，打印日志。



#### **🧠 Week 2：动态大脑注入 —— Go-Lua 沙箱（Day 8 - Day 14）**

**核心目标：** 攻克项目的最大技术难点。让 Go 能够调用 Lua，并把网络请求的能力赋予 Lua。

* **📚 理论弹药：**
* `【核心补充】gopher-lua 库基础与 Go-Lua 相互调用`
* `你需要的那些go语言json技巧`（用于 Go 和 Lua 之间的数据转换）


* **🛠️ 工程落地（每天干什么）：**
1. 在 `internal/service/lua_sandbox.go` 初始化 `gopher-lua` 状态机。
2. **核心难点：** 编写一个 Go 函数（比如发 HTTP GET 请求），并将其**注册到 Lua 虚拟机中**，让 Lua 可以调用宿主的网络能力。
3. 编写 `scripts/github.lua`：在 Lua 中调用刚才注册的 HTTP 方法抓取 GitHub API，解析出 Star 数和描述，返回给 Go。



#### **✨ Week 3：热重载与业务终极闭环（Day 15 - Day 21）**

**核心目标：** 点亮“热更新”的技能树，跑通一条完整数据流，准备写进简历。

* **📚 理论弹药：**
* `【核心补充】fsnotify 本地文件监听实战`


* **🛠️ 工程落地（每天干什么）：**
1. 在 Go 中引入 `fsnotify`，监听 `scripts/` 目录。
2. 当检测到 `github.lua` 文件被修改并保存时，Go 程序自动在内存中重新加载（Reload）该脚本。
3. **闭环测试（验收时刻）：**
* 向 API 发送抓取 GitHub 任务 -> Go 调用 Lua -> Lua 抓取并解析 -> Go 拿到结果用 GORM 存入 MySQL。
* **高潮演示：** 不关闭 Go 程序，直接修改 `github.lua` 里的解析规则，再次发送抓取任务，确认新规则立刻生效！

### 🏆 5 月 19 日的简历包装话术（提前预览）

当这 3 周跑完，你的简历上可以直接写下这段极具杀伤力的文字，并开始海投大厂：

> **项目名称：Go-Lua 动态规则数据提取引擎 (lua-spider)**
> **核心工作：**
> * 基于 Gin + GORM 构建标准化三层架构后端服务，利用 Viper 与 Zap 实现规范化的配置管理与结构化日志追踪。
> * **【核心亮点】** 采用 `gopher-lua` 在 Go 宿主中嵌入 Lua 虚拟机，将核心抓取与解析规则下放至 Lua 脚本，实现底层引擎与上层业务的彻底解耦。
> * **【核心亮点】** 集成 `fsnotify` 监听本地规则目录，实现爬虫解析规则的**零停机热重载 (Hot Reload)**，极大提升了规则迭代与测试的效率。
